# constellation-forge — PRD

## Problem

A star chart with no mythology yet. The user draws figures by connecting
stars; the forge names what they drew and engraves a three-sentence myth
that feels weirdly apt. Cartography as a creative act, shipped as one
dependency-free HTML file in the repo's testable single-file pattern.

## Goals

1. Drawing a figure and sealing it always yields a legend — name,
   designation, myth — derived deterministically from the figure itself,
   so the same drawing on the same sky forges the same legend.
2. The myth's writing quality reads solemn and engraved, never mad-libs
   silly; figure traits visibly steer the text (loop → crown, many stars →
   pursuit, wide span → river).
3. Zero dependencies, zero build, zero network: works from `file://`.
4. All pure logic unit-testable in Node via the shared logic-block harness.

## Non-goals

- Persistence across reloads, sharing, sound, mobile-specific UI.
- Astronomically real star positions; the sky is seeded fiction.

## Functional requirements

- **FR1 — sky:** a seeded field of ~140 stars with varied magnitude and a
  slight twinkle, generated by `makeSky(seed, w, h, n)`; same seed ⇒
  identical sky.
- **FR2 — thread:** a click snaps to the nearest star within a radius and
  starts a thread; subsequent clicks extend the polyline star-to-star.
  Re-clicking the current last star — or pressing Enter — seals the figure.
  Clicking the thread's first star closes the loop and seals.
- **FR3 — seal & engrave:** on seal, the figure's lines redraw with a slow
  luminous gold sweep, then the engraving appears: forged name, designation
  (e.g. "δ Ophelia Borealis"), and a three-sentence myth on a serif
  museum-label card. A small name label marks the figure on the chart.
- **FR4 — determinism:** the legend's RNG is seeded from
  `skySeed ^ figureHash(indices)`, where `figureHash` is order-sensitive.
- **FR5 — name forge:** `forgeName(rng)` produces a capitalized,
  pronounceable 2–4-syllable name (no runs of 3+ consonants).
- **FR6 — myth forge:** `forgeMyth(rng, traits, name)` returns exactly
  three solemn sentences mentioning the name. Trait steering: closed
  figures always draw from the crown/ring family; wide-span figures from
  the river family; many-star figures from the long-pursuit family.
- **FR7 — accumulation:** sealing does not clear the chart; a new thread
  can start on the same sky and multiple engraved figures accumulate.
- **FR8 — keys:** `Enter` seals, `Escape` abandons the current thread,
  `n` forges a fresh sky (new seed, engravings cleared), `s` saves a PNG
  of chart + caption.
- **FR9 — testability:** `mulberry`, `makeSky`, `nearestStar`,
  `figureHash`, `figureTraits`, `forgeName`, `forgeMyth`, and
  `forgeLegend` are exported from `<script id="logic">` via
  `globalThis.__logic` (IIFE-wrapped per the harness convention).

## Quality requirements

- Aesthetic: parchment-on-night — faded-gold chart lines, glinting seeded
  stars, serif caption cards like museum labels, slow luminous line sweep.
- Unit tests deterministic and fast (<10 s); no network in app or tests.
- No console errors in a headless Chromium run from `file://`.

## Acceptance

Headless Playwright run: click ~5 stars to connect, seal with Enter →
name + designation + 3-sentence myth card appears; draw and seal a second
figure on the same sky → both engravings remain; `n` → fresh sky with
engravings cleared; zero page or console errors throughout.
